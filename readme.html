Directory Structure
Sample pages

Numbered html files are sample pages of ePub content

  • 01-section-cover.html
  • 04-sample-content.html
Fonts folder

Contains icon fonts used across publications

Images folder

Page content imagery only

UI folder

Graphics used in epub theme. Custom panels, section covers etc.

SASS

Contains all the SASS partials for CSS compliation. These are not required for final deployment

Stylesheets

Contains compressed version of the CSS stylesheet

javascripts

Contains the Foundation framework, jQuery, modernizer and Cambridge file for custom jQuery.

Syntaxhighlighter is only required for code snippets in elements.html and can be deleted on final deployment

Snippets

HTML with code snippets for insertion into elements.html. This folder can be deleted fror final deployment

Note: Code snippets were previoulsy being done within the document itself but after adding some jQuery to minmise required markup it was interfering with the output, so they were moved into an iframe

Gemfile + Gem.lock

The gem file contains project dependencies and versions. See bundler section below

Bundler

These ePubs are produced using SASS + Compass on the Foundation framework. They also use Bundler to manage any gem dependencies and version that may be required. To make changes and re-compile the SASS use bundle exec compass watch

Any required gems will be installed before you can continue. See http://bundler.io/ for more info

Elements page

The elements page contains examples of the ePub markup with code snippets in order to asisst content production. It is not required for final deployment and can be removed when not its no longer needed.

It does contain dependencies from the Foundation library and other javascript libraries in order to display the content correctly. These can also be deleted when the elements.html page is no longer needed.

Dependencies

The elements page requires the script declaration to import both jQuery and Zepto libraries. No other pages has this.

It uses a javascript library called syntaxhighlighter for code highlighting. It is located in the /javascripts folder and can b removed for deployment

There is also a directory called snippets which contain the actual code snippets which are pulled into the elements page. These are not required for final deployment and can also be deleted.
